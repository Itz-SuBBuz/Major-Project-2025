Project Overview

The Smart Fan Control System is an IoT-based automation project designed to control a DC fan wirelessly through a web interface accessible on any smartphone, tablet, or computer. Using the ESP8266 microcontroller, the system hosts a local Wi-Fi web server that allows users to turn the fan ON/OFF, adjust its speed, and control its tilt angle in real time.

The fan's speed is regulated by a BTS7960 motor driver using the PWM (Pulse Width Modulation) technique, while a servo motor manages the tilting mechanism for airflow direction. The entire setup is powered by an SMPS power supply, and both the ESP8266 and Arduino Nano work together ‚Äî the ESP8266 handles network communication and web control, while the Nano manages the actual motor control signals.

This project demonstrates the integration of IoT with traditional electrical systems to create energy-efficient, user-friendly, and smart home appliances that enhance comfort and convenience.

Objective

The primary objective of the Smart Fan Control System is to design and develop a smart, automated fan that can be wirelessly operated using IoT technology. The project focuses on achieving reliable and smooth fan control through a mobile or web interface.

Circuit Design & Working Principle

Working Principle

The working of this system is based on user interaction through a web interface that communicates with the ESP8266 microcontroller, which generates a PWM (Pulse Width Modulation) signal to control the DC motor speed via the BTS7960 motor driver.

At first, the user connects to the web interface hosted by the ESP8266. The interface provides control buttons such as ON/OFF and speed levels (50%, 75%, 100%). When the user selects any control option, a corresponding command is sent to the ESP8266 through the network.

The ESP8266 receives this input and processes it to generate a PWM signal with a specific duty cycle. This PWM signal represents the average voltage that will be applied to the motor. A higher duty cycle corresponds to higher speed, while a lower duty cycle reduces the motor speed.

The generated PWM signal is then fed into the BTS7960 motor driver module, which amplifies the control signal to a higher current suitable for driving the motor. The BTS7960 acts as a bridge between the low-power control circuit and the high-power motor, ensuring safe and efficient operation.

Finally, the BTS7960 outputs the controlled voltage and current to the DC motor according to the PWM duty cycle. This allows smooth and efficient speed variation of the fan, with quick response to user inputs through the web interface.

Software & Programming

üß© Microcontroller Programming

1Ô∏è‚É£ ESP8266 (NodeMCU) Programming

The ESP8266 hosts a web server accessible from any browser on mobile, tablet, or laptop. It uses Wi-Fi connectivity to communicate with the user and sends real-time commands to the Arduino Nano.

Key Functions:

  • toggleFan() ‚Üí Switches the fan ON/OFF
  • updateSpeed(value) ‚Üí Updates the fan speed via PWM
  • toggleServo(state) ‚Üí Controls the head tilt servo

The PWM value is calculated using map(fanSpeed, 0, 100, 0, 1023) to match the ESP8266 PWM range.

2Ô∏è‚É£ Arduino Nano Programming

Acts as the hardware controller, converting low-power signals from ESP8266 into 5V-compatible PWM signals for the BTS7960 motor driver.

Reads PWM input from ESP8266 using analogRead() and maps it to Nano's PWM output range using map() function. Handles the servo motor movement between 60¬∞ and 120¬∞ using millis() for smooth, non-blocking operation.

3Ô∏è‚É£ Communication & Synchronization

The ESP8266 handles the user interface, network communication, and signal generation. The Arduino Nano performs real-time control of fan speed and servo movement. This separation allows modular programming and easy troubleshooting.

4Ô∏è‚É£ Programming Best Practices Applied
  • Modular functions: Each control task is handled in its own function
  • Mapping & constraining values: Prevents invalid PWM values
  • Non-blocking servo control: Using millis() avoids delays
  • Real-time monitoring: Serial.print() for debugging
ESP8266 Code 
#include <ESP8266WiFi.h>
#include <ESP8266WebServer.h>

const char* ssid = "ssid";        
const char* password = "password"; 

ESP8266WebServer server(80);

bool fanState = false;
int fanSpeed = 75;  // Default speed (used only after turning ON)
bool servoState = false;

const int fanPin = D1;   
const int servoPin = D2; 

String buildHTML() void setup() {
  Serial.begin(115200);
  pinMode(fanPin, OUTPUT);
  analogWriteRange(1023);
  analogWriteFreq(5000);
  pinMode(servoPin, OUTPUT);
  digitalWrite(servoPin, LOW);

  WiFi.begin(ssid, password);
  Serial.print("Connecting to Wi-Fi");
  while (WiFi.status() != WL_CONNECTED) { delay(500); Serial.print("."); }
  Serial.println("\n‚úÖ Connected!");
  Serial.print("IP Address: "); Serial.println(WiFi.localIP());

  server.on("/", []() { server.send(200, "text/html", buildHTML()); });

  server.on("/fan/on", []() {
    fanState = true;
    fanSpeed = 75;  // Default 75% when turned ON
    int pwmVal = map(fanSpeed, 0, 100, 512, 1023);
    analogWrite(fanPin, pwmVal);
    Serial.println("FAN: ON (Default 75%)");
    server.send(200, "text/plain", "OK");
  });

  server.on("/fan/off", []() {
    fanState = false;
    analogWrite(fanPin, 0);
    servoState = false;
    digitalWrite(servoPin, LOW);
    Serial.println("FAN: OFF + SERVO LOCK");
    server.send(200, "text/plain", "OK");
  });

  server.on("/speed", []() {
    fanSpeed = server.arg("val").toInt();
    fanSpeed = constrain(fanSpeed, 0, 100);
    if (fanState) {
      int pwmVal = map(fanSpeed, 0, 100, 512, 1023);
      analogWrite(fanPin, pwmVal);
    }
    Serial.print("SPD: "); Serial.println(fanSpeed);
    server.send(200, "text/plain", "OK");
  });

  server.on("/servo", []() {
    servoState = (server.arg("state") == "on");
    digitalWrite(servoPin, servoState ? HIGH : LOW);
    Serial.print("SERVO: "); Serial.println(servoState ? "ON" : "OFF");
    server.send(200, "text/plain", "OK");
  });

  server.begin();
  Serial.println("HTTP server started");
}

void loop() {
  server.handleClient();
}
Arduino Nano Code 
#include <Servo.h>

const int fanPWMInput = A0;
const int fanPWMOut = 5;
const int motorL = 6; 
const int motorR = 7;
const int servoControlPin = 2;
const int servoPin = 9;

Servo tiltServo;
int servoAngle = 90;
bool servoDirection = false;
unsigned long lastServoTime = 0;
const int servoSpeedDelay = 20;

void setup() {
  Serial.begin(9600);
  pinMode(fanPWMOut, OUTPUT);
  pinMode(motorL, OUTPUT);
  pinMode(motorR, OUTPUT);
  pinMode(servoControlPin, INPUT);
  tiltServo.attach(servoPin);
  tiltServo.write(servoAngle);
}

void loop() {
  int pwmVal = analogRead(fanPWMInput);
  int pwmOut = map(pwmVal, 0, 682, 0, 255);
  pwmOut = constrain(pwmOut, 0, 255);
  analogWrite(fanPWMOut, pwmOut);
  digitalWrite(motorL, HIGH);
  digitalWrite(motorR, HIGH);

  bool servoActive = digitalRead(servoControlPin);

  if (servoActive) {
    unsigned long now = millis();
    if (now - lastServoTime >= servoSpeedDelay) {
      lastServoTime = now;
      if (!servoDirection) {
        servoAngle++;
        if (servoAngle >= 120) servoDirection = true;
      } else {
        servoAngle--;
        if (servoAngle <= 60) servoDirection = false;
      }
      tiltServo.write(servoAngle);
    }
  } else {
    if (servoAngle < 90) servoAngle++;
    else if (servoAngle > 90) servoAngle--;
    tiltServo.write(servoAngle);
  }
}

Features

üì∂ Wi-Fi Enabled Control System

Control your fan over Wi-Fi using any browser on your phone or computer. ESP8266 connects to your home Wi-Fi network and hosts a mini web server. No physical switches needed; control your fan remotely.

üîò Fan ON/OFF Toggle

A large circular button on the webpage allows you to turn the fan on or off. The button changes color for visual feedback ‚Äî Blue for ON and Red for OFF. Instant response for easy operation.

‚ö° Fan Speed Control (PWM)

The speed of the fan can be set among three options: 50%, 75%, and 100%. The ESP converts this percentage into PWM signal (511 for 50%, 767 for 75%, 1023 for 100%) for smooth speed adjustment.

üîÑ Servo (Fan Head Tilt)

Toggle a servo motor to adjust fan head tilt using the "Head Tilt" switch on the webpage. ESP receives commands and powers the servo accordingly for automatic tilting capability.

üìä Real-time Status Display

The webpage shows current Fan State, Tilt State, and Speed. Updates happen using JavaScript after any action, so users see changes immediately without refreshing the page.

üì± Mobile-Friendly Web Interface

Designed to be responsive on smartphones and tablets. CSS ensures layout scales for smaller screens with touch-friendly buttons, sliders, and switches.

üåø Energy-Efficient Control

PWM works by rapidly switching the motor's supply voltage ON and OFF at high frequency. The motor only receives the exact amount of power needed, consuming less power and producing less heat.

üõ°Ô∏è Safety Defaults

When turning off the fan, speed is set to zero and servo is locked to a safe position. This prevents unexpected movement or unnecessary power draw.
